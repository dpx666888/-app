# 🦀 螃蟹记账 — 项目报告

## 项目概述

螃蟹记账是一款跨平台个人财务管理应用，支持 Android/iOS/H5/微信小程序四大平台。提供快速记账、月度统计、日历账单、Excel/CSV/JSON 数据导出导入等完整功能链，帮助用户清晰掌握每一笔收支。

| 项目 | 内容 |
|------|------|
| 项目名称 | 螃蟹记账 |
| 版本号 | 1.2.2 |
| 作者 | 大螃蟹 |
| 技术框架 | uni-app (Vue 3 Options API) |
| 目标平台 | Android APK / iOS / H5 浏览器 / 微信小程序 |
| 代码行数 | ~4000+ (Vue SFC + JS 工具模块) |
| 页面数量 | 7 个页面（含 5 个 Tab 页） |

## 页面功能详述

### 1. 首页 (index)
- 用户问候 + 日期展示（早上好/下午好/晚上好）
- 本月概览卡片：总支出 / 总收入 / 结余
- 最近记录列表（分页加载，每次 10 条）
- **悬浮记账按钮**：可随意拖拽的 FAB，点击快速跳转记账
- 未登录时展示欢迎引导页 + 功能预览

### 2. 记账页 (add)
- 支出/收入类型切换
- 金额输入（自动限制小数点后两位）
- 9 种支出分类：餐饮、交通、购物、娱乐、医疗、教育、住房、通讯、其他
- 4 种收入分类：工资、奖金、投资、其他
- 可选备注，保存后自动跳转首页

### 3. 统计页 (report)
- 月份选择器（仅显示有数据的月份）
- 月度收支汇总卡片
- 分类分布图（进度条 + 百分比）
- 明细记录列表（切换支出/收入时自动过滤）
- 长按记录编辑/删除
- 导出当月 Excel

### 4. 日历页 (calendar)
- 月份导航（前后翻月）
- 日历网格（支出/收入用红绿圆点标记）
- 点击日期查看当日汇总（总支出/总收入）
- 当日记录列表，长按编辑/删除
- 未登录引导

### 5. 个人中心 (profile)
- 头像上传（拍照/相册选取）
- 修改密码
- 多用户管理（一键切换）
- 数据导出（Excel / CSV / JSON 备份，三选一菜单）
- JSON 备份导入（覆盖模式，需确认）
- 一键清除当前用户数据
- 退出登录

## 数据存储设计

### 存储方式
- **存储引擎**：`uni.setStorageSync` / `uni.getStorageSync`（本地 localStorage 封装）
- **无后端**：全部数据存储在客户端本地，无需网络、无需服务器
- **存储限制**：受浏览器/WebView 的 localStorage 配额限制（通常 5~10MB）

### 数据库类型
- **类型**：无数据库，使用 **JSON 文件序列化存储**（localStorage key-value）
- **查询方式**：全量加载后 JS 内存中 filter/sort/reduce 操作
- **持久化**：Android 平台存储于应用沙盒，iOS 存储于 NSUserDefaults，浏览器存储于 localStorage

### 数据模型

**users (用户列表)**
```json
{
  "id": "唯一ID (时间戳36进制+随机数)",
  "username": "用户名",
  "password": "明文密码",
  "avatar": "头像文件路径或Base64",
  "records": [
    {
      "id": "记录唯一ID",
      "type": "expense | income",
      "amount": 100.00,
      "category": "分类名称",
      "note": "备注文字",
      "createTime": "ISO 8601 时间字符串",
      "showTime": true    // 是否显示具体时间
    }
  ]
}
```

**存储键名**
| Key | 类型 | 说明 |
|-----|------|------|
| `users` | JSON string | 全部用户数组 |
| `current_user` | JSON string | 当前登录用户完整对象 |

### 数据流

```
记账页 (add) → addRecord() → 更新 users 数组 → setStorageSync → 首页/统计/日历 onShow → getRecords() → 渲染更新
```

无响应式状态管理库（无 Vuex/Pinia），依赖 uni-app 页面的 `onShow` 生命周期每次刷新全量数据。

### 多平台导出实现差异

| 平台 | Excel/CSV 导出 | JSON 备份导出 | JSON 备份导入 |
|------|---------------|--------------|--------------|
| **Android (APP-PLUS)** | `Intent.ACTION_CREATE_DOCUMENT` 系统文件选择器保存 | 同上 | `ContentResolver.openInputStream` 逐字节读取 |
| **iOS** | 同上（5+App 统一 API） | 同上 | 同上 |
| **H5** | `Blob + URL.createObjectURL + <a>` 触发下载 | 同上 | `FileReader.readAsText` |
| **微信小程序** | `FileSystemManager.writeFile` 写入临时文件 | `wx.shareFileMessage` 分享 | `wx.chooseMessageFile` 选择文件 |

## 性能优化策略

- **大数据量分批导出**：记录数 > 3000 时启用异步分批处理，每 2000 条让出主线程
- **日期格式化缓存**：使用 `Map` 缓存已格式化的日期字符串，避免重复计算
- **数组预分配**：导出时使用 `new Array(len)` 预分配数组，避免动态扩容
- **Excel 写入优化**：禁用 `bookSST`（共享字符串表）和 `compression`，减少内存占用
- **异步让出主线程**：`yieldToMain()` 通过 `setTimeout(0)` 让出事件循环，防止 UI 卡死

## 安全说明

- 密码以明文存储于 localStorage（无哈希加密）
- 无网络请求，数据不出设备（隐私性好但无云备份）
- 无用户认证 token、无会话管理（基于本地存储的用户对象判断登录态）
- Android 权限清单包含相机、存储读写等标准权限

## 局限性 / 可改进方向

| 问题 | 说明 |
|------|------|
| 密码明文存储 | 建议加盐哈希 (bcrypt/sha256) 后再存储 |
| 无云同步 | 数据仅存本地，换设备或卸载即丢失 |
| 无 Pinia/Vuex | 页面间数据靠 onShow 重新读取，非响应式共享 |
| 无单元测试 | 全部功能未编写自动化测试 |
| 无 TypeScript | 全程 JS 弱类型，大型改动易出运行时错误 |
| 无国际化 | 仅支持简体中文 |
| 图表简陋 | 使用纯 CSS 进度条而非 ECharts 等专业图表库 |
