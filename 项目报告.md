# 🦀 螃蟹记账 — 项目报告

## 项目概述

螃蟹记账是一款跨平台个人财务管理应用，支持 Android/iOS/H5/微信小程序四大平台。提供快速记账、ECharts 图表统计、gzip 压缩备份、Excel/CSV 数据导出等完整功能链，帮助用户清晰掌握每一笔收支。

| 项目 | 内容 |
|------|------|
| 项目名称 | 螃蟹记账 |
| 版本号 | 0.3.2 |
| 作者 | 大螃蟹 |
| 技术框架 | uni-app (Vue 3 Options API + Pinia) |
| 目标平台 | Android APK / iOS / H5 浏览器 / 微信小程序 |
| 代码行数 | ~6000+ (Vue SFC + JS 工具模块) |
| 页面数量 | 6 个页面（含 4 个 Tab 页） |

## 页面功能详述

### 1. 首页 (index)
- 用户问候 + 日期展示（早上好/下午好/晚上好）
- 本月概览卡片：总支出 / 总收入 / 结余
- 最近记录列表（分页加载，每次 10 条）
- **悬浮记账按钮**：可拖拽移动（CSS transform 偏移，避免重排），点击跳转记账
- 未登录时展示欢迎引导页 + 功能预览

### 2. 记账页 (add)
- 支出/收入类型切换
- 金额输入（自动限制小数点后两位）
- 9 种支出分类：餐饮、交通、购物、娱乐、医疗、教育、住房、通讯、其他
- 4 种收入分类：工资、奖金、投资、其他
- 可选备注
- **保存成功浮层**：保存后弹出浮层，可选择"再记一笔"继续记账或"取消"返回首页，4 秒自动返回

### 3. 统计页 (report)
- 月份选择器 + 周/月视图切换
- 月度收支汇总卡片
- **ECharts 图表**（renderjs 渲染）：
  - 饼图：分类金额占比，按支出/收入切换
  - 柱状图：每日收支对比，渐变配色 + 圆角
  - 折线图：收支趋势，带渐变面积填充
- **时间范围筛选**（柱/折线图）：今天 / 近7天 / 近15天 / 整月
- 明细记录列表，长按编辑/删除
- 底部固定导出按钮

### 4. 个人中心 (profile)
- 头像上传（拍照/相册选取）
- 修改密码
- 多用户管理（一键切换）
- 数据导出（Excel / CSV / JSON gzip 备份，三选一菜单）
- JSON 备份导入（自动识别旧格式或 gzip 压缩格式）
- 一键清除当前用户数据
- 退出登录

## 数据存储设计

### 存储方式
- **存储引擎**：`uni.setStorageSync` / `uni.getStorageSync`（本地 localStorage 封装）
- **无后端**：全部数据存储在客户端本地，无需网络、无需服务器
- **存储限制**：受浏览器/WebView 的 localStorage 配额限制（通常 5~10MB）
- **存储拆分**：用户数据与记录数据分离存储（`userStorage.js` / `recordStorage.js`）

### 数据模型

**users (用户列表)**
```json
{
  "id": "唯一ID (时间戳36进制+随机数)",
  "username": "用户名",
  "passwordHash": "加盐哈希后的密码",
  "salt": "随机盐值",
  "avatar": "头像文件路径或Base64"
}
```

**records (记录列表，独立存储)**
```json
{
  "id": "记录唯一ID",
  "userId": "所属用户ID",
  "type": "expense | income",
  "amount": 100.00,
  "category": "分类名称",
  "note": "备注文字",
  "createTime": "ISO 8601 时间字符串"
}
```

**存储键名**
| Key | 类型 | 说明 |
|-----|------|------|
| `crab_users` | JSON string | 全部用户数组 |
| `crab_records` | JSON string | 全部记录数组 |
| `crab_current_user` | JSON string | 当前登录用户完整对象 |

### 数据流

```
记账页 (add) → Pinia store → addRecord() → recordStorage.saveAllRecords()
  → 响应式更新首页/统计页 (无需 onShow 全量重读)
```

引入 Pinia 后告别了全量 `onShow` 刷新，页面间数据为响应式共享。

### 多平台导出实现差异

| 平台 | Excel/CSV 导出 | JSON 备份导出 | JSON 备份导入 |
|------|---------------|--------------|--------------|
| **Android (APP-PLUS)** | `Intent.ACTION_CREATE_DOCUMENT` 系统文件选择器保存 | 同上 | `ContentResolver.openInputStream` 逐字节读取 |
| **iOS** | 同上（5+App 统一 API） | 同上 | 同上 |
| **H5** | `Blob + URL.createObjectURL + <a>` 触发下载 | 同上（gzip base64） | `FileReader.readAsArrayBuffer` |
| **微信小程序** | `FileSystemManager.writeFile` 写入临时文件 | `wx.shareFileMessage` 分享 | `wx.chooseMessageFile` 选择文件 |

## 性能优化策略

- **ECharts renderjs 渲染**：图表在独立渲染层运行，不阻塞 UI 线程
- **FAB transform 偏移**：拖拽时使用 CSS transform 而非修改 left/top，避免重排
- **ECharts 动画**：图表启用 600ms 动画，过渡流畅
- **备份 gzip 压缩**：导出数据压缩后 base64 编码，体积减少约 70%
- **大数据量分批导出**：记录数 > 3000 时启用异步分批处理，每 2000 条让出主线程
- **日期格式化缓存**：使用 Map 缓存已格式化的日期字符串，避免重复计算
- **数组预分配**：导出时使用 `new Array(len)` 预分配数组，避免动态扩容
- **Excel 写入优化**：禁用 `bookSST` 和 `compression`，减少内存占用

## 安全说明

- 密码以 **crypto-js 加盐哈希** 存储（非明文），提高密码安全性
- 无网络请求，数据不出设备（隐私性好但无云备份）
- 无用户认证 token、无会话管理（基于本地存储的用户对象判断登录态）
- Android 权限清单包含相机、存储读写等标准权限
